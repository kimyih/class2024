01.if문


if문은 프로그래밍에서 조건을 판단하고, 조건이 참일 때 특정 코드를 실행하는 제어 구조입니다.
if문은 주어진 조건이 참인지 거짓인지를 판단하여 프로그램이 다른 동작을 하도록 합니다.
일반적으로 조건이 참이면 if문 다음에 오는 중괄호({}) 안의 코드가 실행되고, 거짓이면 해당 코드는 실행되지않습니다.
else문을 사용하여 조건이 거짓일 때 실행할 코드 블록을 지정할 수 있습니다.
또한, else if문을 사용하여 여러 조건을 검사하고 다양한 경우에 대응할 수 있습니다.
이러한 if문은 프로그램이 다양한 상황에 대응하고 특정 조건에 따라 다른 동작을 수행할 수 있도록 도와줍니다. 

{
    // true, "문자열" 1, 2, "1", "2", [], {}
    // false. ""(빈문자열), 0, null, undefined  ->flase가 나오는 건 이 다섯가지밖에 없음 

    if (0) {
        console.log("실행되었습니다.(true)");
    } else {
        console.log("실행되었습니다.(false)");
    }
}

02. 다중 if문


다중 if문은 여러 개의 조건을 순차적으로 검사하며, 각 조건에 따라 다른 코드를 실행하는 제어 구조입니다.
각각의 if문은 이전 조건이 거짓일 때 실행되며, 조건이 참일 때 해당 코드 블록을 실행합니다.
조건이 거짓이면 다음 if문이 검사됩니다.
마지막에는 else문을 사용하여 모든 조건이 거짓일 때 실행할 코드를 지정할 수 있습니다.
다중 if문은 복잡한 조건 로직을 처리하고 다양한 상황에 대응하기 위해 사용됩니다.
그러나 너무 많은 중첩은 코드를 이해하기 어렵게 만들 수 있으므로 필요할 때만 사용하는 것이 좋습니다. 

{
    const num = 100;

    if (num == 90) {
        console.log("실행되었습니다.(90)");
    } else if (num == 100) {
        console.log("실행되었습니다.(100)");
    } else if (num == 110) {
        console.log("실행되었습니다.(110)");
    } else if (num == 120) {
        console.log("실행되었습니다.(120)");
    } else if (num == 130) {
        console.log("실행되었습니다.(130)");
    } else {
        console.log("실행되었습니다.");
    }
}
결과 확인하기
실행되었습니다.(100)

03. 중첩 if문


중첩 if문은 하나의 if문 안에 또 다른 if문을 포함하여, 조건에 따라 중첩된 코드 블록을 실행하는 제어 구조입니다.
이는 프로그램이 여러 조건을 평가하고, 해당하는 조건에 따라 다른 동작을 수행할 수 있도록 해줍니다.
외부 if문의 조건이 참이면 내부 if문이 검사되고, 내부 if문의 조건이 참이면 해당하는 코드 블록이 실행됩니다.
중첩 if문은 복잡한 조건 로직을 구현하고 다양한 상황에 대응하기 위해 사용됩니다.
그러나 과도한 중첩은 코드를 이해하기 어렵게 만들 수 있으므로 필요할 때만 사용하는 것이 좋습니다. 

{
    const num = 100;

    if (num == 100) {
        console.log("실행되었습니다.1");
        if (num == 100) {
            console.log("실행되었습니다.2");
            if (num == 100) {
                console.log("실행되었습니다.3")
            }
        }
    } else {
        console.log("실행되었습니다.4")
    }
}

04. if문 생략 & 삼항 연산자


삼항 연산자는 조건을 평가하여 참일 때와 거짓일 때 다른 값을 반환하는 연산자입니다.
조건식이 참이면 '?' 뒤에 있는 값이 반환되고, 거짓이면 ':' 뒤에 있는 값이 반환됩니다.
이를 통해 if문을 간결하게 표현할 수 있으며, 코드를 간단하고 가독성 있게 작성할 수 있습니다. 주로 간단한 조건을 다룰 때 사용되며, 프로그램의 유연성을 높이고 코드의 가독성을 높여줍니다. 

{
    const num = 100;

    if (num == 100) {
        console.log("true")
    }

    // 생략
    if (num == 100) console.log("ture")  //생략 가능 자바스크립트에선 생략 잘 안 함 가독성 떨어져서 파이썬에선 사용 많이 함

    if (num == 100) {
        console.log("true");
    } else {
        console.log("false");
    }
    //삼항 연산자 (조건식 연산자);
    num == 100 ? console.log("true") : console.log("false");
}

05. switch문


switch문은 프로그래밍 언어에서 조건에 따라 여러 경우를 처리하는 제어 구조입니다.
변수나 표현식을 기반으로 여러 case로 분기되며, 각 case는 특정 값에 대응합니다.
각 case는 실행 후 break 문으로 종료되며, default는 모든 case에 해당하지 않을 때 실행될 코드를 정의합니다.
여러 값을 하나의 case에 지정할 수 있으며, 일부 언어에서는 case 레이블 사이에 break가 없으면 fall-through 동작이 발생합니다.

{
    const num = 100;

    switch (num) {
        case 90:
            console.log("실행되었습니다.(90)");
            break;
        case 95:
            console.log("실행되었습니다.(95)");
            break;
        case 100:
            console.log("실행되었습니다.(100)");
            break;
        case 105:
            console.log("실행되었습니다.(105)");
            break;
        default:

            console.log("실행되었습니다.");
    }
}

실행되었습니다.(100)

06. while문


While문은 주어진 조건이 참일 때에만 특정 코드를 반복적으로 실행하는 제어 구조입니다.
조건이 거짓이 될 때까지 코드 블록을 실행하며, 반복문을 종료하려면 조건을 거짓으로 만들어야 합니다.
주로 반복 횟수가 명확하지 않을 때 사용되며, 무한루프에 빠지지 않도록 주의해야 합니다.
While문을 사용할 때는 종료 조건을 명확히 설정하여야 하며, 무한 반복에 주의해야 합니다. 

{
        let num = 1;        // 초깃값;

        while (num <= 10) {     // 조건식;
            console.log(num);   // 실행문
            num++;      //증감값    for문과 while문의 차이는 순서 차이 어떤 거 먼저 실행시키냐 차이
        }
}

07. do while문


Do while문은 프로그래밍에서 반복문을 사용할 때 조건을 검사하는 위치가 다른 while문과는 다르게, 먼저 코드 블록을 실행한 후에 조건을 확인합니다.
이는 최소 한 번은 코드 블록을 실행해야 할 때 유용합니다.
코드 블록이 먼저 실행되므로 반복 실행 여부와는 관계없이 최소한 한 번은 코드가 실행됩니다.
따라서 사용자 입력 처리나 초기화 코드를 실행할 때 유용하게 사용됩니다.
또한, 반복문을 빠져나오기 위한 조건이 조건 검사 이전에 먼저 실행되어야 할 때 사용될 수 있습니다.
그러나 반복문이 조건을 만족하지 않아도 최소 한 번은 실행되므로, 주의가 필요합니다.

{
    let num = 1; // 초깃값;

    do {
        console.log(num);   // 실행문
        num++;      // 증감값
    } while (num <= 10);     // 조건식
}

08. for문


for문은 프로그래밍에서 반복 작업을 처리하는 제어 구조 중 하나로, 초기화, 조건 검사, 증감식을 한 줄에 나열하여 사용합니다.
초기화는 반복문 시작 전 한 번 실행되며, 조건 검사는 각 반복마다 실행되어 조건이 참일 경우 반복을 계속하고 거짓일 경우 종료합니다.
증감식은 각 반복이 끝난 후 실행되어 변수 값을 변경합니다.
for문은 반복 횟수가 명확할 때 주로 사용되며, 배열이나 리스트와 같은 자료구조를 순회할 때 효율적으로 사용됩니다. 코드의 가독성이 높고 구조가 간결하여 많이 활용됩니다.

{
        for (let i = 1; i <= 10; i++) {     // 초깃값; 조건식; 증감값 
            console.log(i)      // 실행문
    }
}

09. 중첩for문


중첩 for문은 한 for문 안에 다른 for문이 포함된 구조로, 반복 작업을 다양한 방식으로 처리할 때 사용됩니다.
바깥쪽 for문의 각 반복마다 안쪽 for문이 전체 반복되며, 이러한 과정이 바깥쪽 for문의 반복 횟수만큼 반복됩니다.
주로 2차원 배열과 같은 다차원 데이터 구조를 순회하거나, 이차원 공간을 탐색하는 데 사용됩니다.
중첩 for문을 사용하면 복잡한 패턴을 처리하거나 다양한 문제를 해결할 수 있습니다.
그러나 중첩 for문은 코드의 복잡성을 증가시킬 수 있으므로 적절한 사용이 필요합니다. 

{
    for (let i = 1; i <= 10; i++) {     // 초깃값; 조건식; 증감값 
        console.log(i)      // 실행문
        for (let j = 1; j <= 10; j++) {     // 초깃값; 조건식; 증감값 
            console.log(j)      // 실행문
        }
    }
}

10. break문


break문은 프로그래밍에서 반복문이나 switch문에서 실행을 중단하고 반복문 또는 switch문을 빠져나오는데 사용됩니다.
반복문 안에서 break문이 실행되면 반복문의 실행이 즉시 중단되고, 해당 반복문 블록을 벗어납니다.
또한 switch문 안에서 break문이 실행되면 switch문을 빠져나오게 됩니다.

일반적으로 반복문에서 break문은 특정 조건을 만족할 때 루프를 종료하거나, 반복문을 빠져나가야 하는 경우에 사용됩니다.
예를 들어, 반복문 안에서 특정 조건을 만족하면 반복을 중단하고 빠져나가야 할 때 break문을 사용합니다.
또한 switch문에서는 각 case문을 실행한 후에 반드시 break문을 사용하여 switch문을 종료해야 합니다.
그렇지 않으면 다음 case문이 연이어 실행될 수 있습니다.

break문은 프로그램의 흐름을 제어하는데 유용하며, 특정 조건에 따라 루프를 종료하거나 switch문을 빠져나가는데 활용됩니다.
그러나 break문을 남용하면 코드의 가독성이 떨어지고, 프로그램의 동작을 이해하기 어려워질 수 있으므로 신중하게 사용해야 합니다.


    for (let i = 1; i <= 10; i++) {
        if (i == 5) {
            break;
        }
        console.log(i)
    }
}

11. continue문


continue문은 반복문에서 사용되며, 실행 중에 continue문을 만나면 해당 반복의 나머지 부분을 건너뛰고 다음 반복을 진행합니다.
즉, continue문이 실행되면 반복문의 현재 반복은 중지되고, 다음 반복이 시작됩니다.

일반적으로 continue문은 특정 조건을 만족할 때 특정 반복을 건너뛰고자 할 때 사용됩니다.
예를 들어, 반복문 안에서 특정 조건을 만족하는 경우에는 해당 반복을 실행하지 않고 건너뛰고자 할 때 continue문을 사용합니다.

continue문은 주로 반복문 안에서 특정 조건을 검사하여 특정 작업을 수행하지 않고 다음 반복으로 넘어갈 때 사용됩니다.
이는 반복 중간에 특정 조건이 발생할 경우 다음 반복을 실행하고자 할 때 유용하게 활용됩니다. 

{
    for (let i = 1; i <= 10; i++) {
        if (i == 5) {
            continue;
        }
        console.log(i)
    }
}
